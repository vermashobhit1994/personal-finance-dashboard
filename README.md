# Introduction

## Problem

- User lack necessary `visibility` and automated tools to align their `spending habits` with their long-term `financial goals`.

Example - Users are forced to aggregate fragmented data from financial apps( for 
eg phonepay, googlePay, YonoSbi) which causes high cognitive overhead to
categorise manual transactions and also inaccurate understanding of their spending habits which prevents user from maintaining consistent long-term budget.

## Solution
Built end to end **`AI Kharcha hisab`** tool (personal finance dashboard) with production build that takes bank statements as data and do analysis then provide 
dashboard to see finances visually

### **`Why to use Tool`**

1. save `time`, `effort`(cognitive overhead), `energy` of person to do analysis of their financial habits
2. Use `AI` to give recommendations for creating `expenses category`, `budget plan`, `wallet`, `income sources` by analysis of **spending habits** and **monthly expenses** and thus increase savings and decrease expenses.



Building AI Kharcha hisab tool includes below stages:

`Stage 1`: gathering user story 

`Stage 2`: gathering and analysis of requirements 

`Stage 3`: website design using UI/UX design principles and design tool.

`Stage 4`: Frontend System design and Backend System Design 

`Stage 5`: website development using framework, tools, library using software engineering best practices and fundamental concepts.

`Stage 6`: create CI/CD pipelines using Vercel

`Stage 7`: Shipped and deployment to production on vercel
